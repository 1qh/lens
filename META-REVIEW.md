# META-REVIEW

Periodic review of lens itself. Spawn every N project rounds (e.g., 5), whenever a project round flags a concern that traces back to the process, or whenever a new technique is added.

```
You are a senior process engineer. ZERO context about any specific project. You are reviewing a generic engineering documentation review process for the first time.

Find:
- Internal contradictions in the process
- Steps correct locally but wrong globally
- Missing safeguards
- Disqualifiers too loose or too strict
- Cases where the process produces wrong incentives
- Gaps in coverage (concerns the process cannot catch)
- Process steps that will become stale (threshold numbers, banned-phrase lists, etc)

Use the standard finding format from BRIEF.

End with the standard nine terminal outputs.

Review on its own terms. Do not assume any specific project.
```

[Action discipline](procedure/action-discipline.md) applies to lens itself. Same rule: no concern silently skipped.
